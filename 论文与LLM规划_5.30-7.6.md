# 5/30-7/6 论文与 LLM 学习规划

这版只管两件事：论文精读和 LLM 课程。  
专业课、四级、期末模拟题时间全部空出来，不在这里细排。

## 0. 总原则

优先级：

1. 专业课和期末复习时间不可占用
2. 四级前英语时间不可占用
3. 论文精读优先于 LLM
4. LLM 只做轻量推进，不追求刷完课程

你当前状态：

- LLM 课程一、二章已学完
- 论文已粗读一遍
- 现在要进入“能讲清楚”的阶段，不是重新从头看

每天判断状态：

| 状态 | 论文/LLM最低任务 |
|---|---|
| 很差 | 不学，最多看 10 分钟笔记 |
| 一般 | 论文或 LLM 选一个，30 分钟 |
| 正常 | 论文 45-60 分钟，或 LLM 30-45 分钟 |
| 很好 | 论文 60-90 分钟 + LLM 30 分钟 |

规则：

- 周四默认不安排论文/LLM。
- 7/1-7/5 考前周基本暂停，只保留极轻复盘。
- 不补债。错过就跳过，不往后堆。

## 2. 论文主线

论文：Beyond the Final Actor: Modeling the Dual Roles of Creator and Editor for Fine-Grained LLM-Generated Text Detection  
链接：https://arxiv.org/abs/2604.04932

核心目标：

- 不是“看完”论文，而是能讲清楚这篇论文解决什么问题、为什么要四分类、RACE 怎么做、实验说明了什么。
- 最终产出：一页中文精读笔记 + 一张方法流程图 + 5 个问题 + 3 个可汇报亮点。

### 5/30-6/2：从粗读变成结构理解

任务：

- 重看 Abstract、Introduction、Figure 1。
- 明确四类文本：
  - 人写
  - 人写后 AI 润色
  - AI 写
  - AI 写后人类改写
- 搞清楚 creator 和 editor 分别是什么意思。

输出：

- 一张四分类表：
  - creator 是谁
  - editor 是谁
  - 文本风险/政策含义是什么
  - 为什么二分类不够

最低档：

- 只写出四类文本和 creator/editor 对应关系。

### 6/3-6/6：相关工作，不要逐篇啃引用

任务：

- 只抓三条线：
  - 传统二分类 AI 文本检测
  - 细粒度 AI 文本检测
  - 人机协作文本检测
- 每条线只回答：
  - 它想解决什么问题
  - 它为什么不够
  - 这篇论文怎么补它的缺口

输出：

- 一个三列表：
  - 研究线
  - 以前怎么做
  - 这篇论文为什么还要做

最低档：

- 只看 Related Work 小节标题和每段第一句，先抓方向。

### 6/7-6/10：方法 RACE

任务：

- 看懂 RACE 的整体流程。
- 重点理解两件事：
  - 用 RST 逻辑结构建模 creator
  - 用 EDU-level 局部特征建模 editor
- 不要求现在完全看懂公式，先看懂每个模块“输入什么、输出什么、为什么要这样做”。

输出：

- 一张方法流程图，至少包含：
  - 输入文本
  - RST/逻辑图
  - EDU 局部特征
  - creator 表征
  - editor 表征
  - 四分类输出

最低档：

- 写 5 句话解释 RACE：
  - 它为什么叫 RACE
  - 它分开建模什么
  - RST 管什么
  - EDU 管什么
  - 最后输出什么

### 6/11-6/16：实验部分

任务：

- 看数据集、baselines、metrics、主结果表。
- 只回答四个问题：
  - 它和哪些 baseline 比
  - 主要指标是什么
  - RACE 强在哪里
  - 有没有明显弱点或风险

输出：

- 一张实验总结表：
  - 数据集
  - 对比方法
  - 指标
  - 主要结论
  - 我的疑问

最低档：

- 只看主结果表和 ablation 表，不钻细节。

### 6/17-6/23：第二轮精读收束

任务：

- 把前面零散笔记合成一页中文总结。
- 整理 5 个真问题，不要写假装高级的问题。
- 整理 3 个可汇报亮点。

输出：

- 一页中文精读笔记：
  - 研究问题
  - 任务设定
  - 方法核心
  - 实验结论
  - 局限/疑问
- 5 个没懂或可讨论的问题
- 3 个汇报亮点

最低档：

- 只完成一页中文总结，问题和亮点之后再补。

### 6/24-6/30：维护，不再扩展

任务：

- 不读新参考文献。
- 不开新方向。
- 只把已有笔记修顺。

输出：

- 论文笔记最终版。
- 如果需要汇报，准备一个 3 分钟口头版：
  - 这篇论文研究什么
  - 方法怎么做
  - 为什么有意义

### 7/1-7/5：考前周暂停

只允许做一件事：

- 每两天看一次论文总结，10-15 分钟。

不允许：

- 读新论文
- 补参考文献
- 重画方法图
- 学新 LLM 章节

## 3. LLM 主线

课程：Lordog/dive-into-llms  
链接：https://github.com/Lordog/dive-into-llms

你已经学完一、二章，所以接下来不从头复习。

### 现在到 6/13：只补和论文有关的 LLM 背景

优先顺序：

1. 模型水印
2. 大模型安全/越狱攻击
3. 知识编辑

原因：

- 你的论文是 LLM-generated text detection。
- 模型水印和检测最相关。
- 安全/越狱能帮你理解“为什么要监管 LLM 生成内容”。
- 知识编辑可以轻量了解，不作为主线。

每次 LLM 学习只做 30-45 分钟，输出三句话：

```text
今天看了：
我能解释：
我没懂：
```

### 6/14-6/30：LLM 降到维护

每周最多 2 次，每次 30 分钟。

只做：

- 复盘一、二章笔记
- 看模型水印/安全相关小节
- 整理和论文相关的概念

不做：

- 数学推理
- 多模态
- GUI Agent
- RLHF
- 大量跑代码

### 7/1-7/5：暂停 LLM

考前周不学新 LLM。

如果手痒，只能做 10 分钟：

- 看之前写的三句话笔记
- 不打开新章节
