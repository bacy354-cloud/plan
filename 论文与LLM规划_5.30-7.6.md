## 4. 每个产出怎么写

### 四分类表

| 类型 | creator | editor | final actor | 你要写清楚什么 |
|---|---|---|---|---|
| Human-written | 人 | 人/无明显编辑 | 人 | 纯人类文本 |
| LLM-polished human text | 人 | LLM | LLM | 人写内容，AI 改表达 |
| LLM-generated text | LLM | LLM/无明显人类编辑 | LLM | AI 主体生成 |
| Humanized LLM text | LLM | 人 | 人 | AI 写底稿，人类改写 |

### RACE 方法图

按这个结构画：

```text
输入文本
  -> RST 篇章结构
  -> creator 表征
  -> EDU 局部文本单元
  -> editor 表征
  -> 融合 creator/editor
  -> 四分类输出
```

每个箭头旁边写一句话：

- RST 看整体逻辑。
- EDU 看局部表达。
- creator 更像“谁搭了文章骨架”。
- editor 更像“谁改了局部表达”。

### 实验总结表

| 项目 | 要填什么 |
|---|---|
| 数据集 | 文本从哪来，四类怎么构造 |
| baselines | 和哪些检测方法比 |
| metrics | 主要指标是什么 |
| 主结果 | RACE 相比 baseline 强在哪里 |
| ablation | 去掉哪个模块影响最大 |
| 局限 | 短文本、跨领域、强改写、数据构造是否有风险 |
