## 0. 使用规则

你的当前状态：

- LLM 课程一、二章已经学完。
- 论文已经粗读一遍。
- 现在目标不是“继续看很多东西”，而是把论文和相关 LLM 背景变成能讲清楚的笔记。

优先级：

1. 专业课/期末复习优先，不能被这份计划挤占。
2. 四级前英语优先，论文和 LLM 只能塞小块时间。
3. 论文优先于 LLM。
4. LLM 只学和论文有关的章节，不刷完整课程。

状态不好时这样降级：

| 状态 | 当天只做什么 |

## 2. 论文精读路线

论文：Beyond the Final Actor: Modeling the Dual Roles of Creator and Editor for Fine-Grained LLM-Generated Text Detection  
链接：https://arxiv.org/abs/2604.04932

核心问题：

- 传统 AI 文本检测常常只问“是不是 AI 写的”。
- 这篇论文要细分 creator 和 editor 两个角色。
- 所以它关心四类文本：人写、人写后 AI 润色、AI 写、AI 写后人类改写。

### 板块 A：结构和任务设定，3h

目标：先把“这篇论文到底在问什么”讲明白。

| 小任务 | 时间 | 具体做法 | 输出 |
|---|---:|---|---|
| Abstract 复读 | 20min | 圈出 task、method、result 三句话 | 论文一句话总结初稿 |
| Introduction 复读 | 40min | 找作者说“二分类不够”的理由 | 问题背景 3 句话 |
| Figure 1 | 30min | 看四类文本的区别，不纠结细节 | 四分类草表 |
| 四分类表 | 60min | 写 creator/editor/风险/例子 | 完整四分类表 |
| 任务设定复述 | 30min | 用自己的话解释 fine-grained detection | 100 字中文解释 |

四分类表模板：

| 类型 | creator | editor | 你要怎么理解 |
|---|---|---|---|
| Human-written | 人 | 人/无明显编辑 | 原始人类文本 |
| LLM-polished human text | 人 | LLM | 人写内容，AI 改表达 |
| LLM-generated text | LLM | LLM/无明显人类编辑 | AI 主体生成 |
| Humanized LLM text | LLM | 人 | AI 写底稿，人类改写 |

### 板块 B：相关工作，3h

目标：不逐篇啃参考文献，只知道这篇论文站在哪些工作后面。

| 小任务 | 时间 | 具体做法 | 输出 |
|---|---:|---|---|
| 二分类检测 | 45min | 只看 Related Work 对二分类检测的概括 | “二分类为什么不够”3 句话 |
| 细粒度检测 | 45min | 找和四分类/多类别检测有关的工作 | 细粒度检测缺口 |
| 人机协作文本 | 45min | 看 AI 润色、人类改写相关讨论 | creator/editor 必要性 |
| 参考文献筛选 | 45min | 只筛 4 篇优先读摘要 | 文献优先级表 |

参考文献优先级：

| 优先级 | 方向 | 怎么读 |
|---|---|---|
| 必看摘要 | DetectGPT / Fast-DetectGPT | 看它们为什么适合二分类或零样本检测 |
| 必看摘要 | LLM-DetectAIve / Beyond Binary | 看它们怎么做细粒度检测 |
| 只查概念 | RST / EDU | 只补方法所需概念 |
| 可跳过 | 大量 benchmark 细节 | 期末前不展开 |

### 板块 C：方法 RACE，5h

目标：不要先死磕公式，先搞清楚每个模块输入什么、输出什么、为什么需要。

| 小任务 | 时间 | 具体做法 | 输出 |
|---|---:|---|---|
| 方法总览 | 30min | 先看方法图/方法段落开头 | RACE 总流程 5 句话 |
| RST 基础 | 60min | 查 RST 是怎么表示篇章逻辑关系的 | RST 概念卡 |
| EDU 基础 | 60min | 搞懂 EDU 是比句子更小的语义单元 | EDU 概念卡 |
| creator 建模 | 75min | 理解为什么 creator 更像“整体逻辑结构” | creator 模块说明 |
| editor 建模 | 75min | 理解为什么 editor 更像“局部表达/风格痕迹” | editor 模块说明 |
| 方法图整理 | 30min | 画一张自己的流程图 | RACE 方法图 |

方法图至少包含：

```text
输入文本
  -> RST / 篇章逻辑结构
  -> creator 表征
  -> EDU-level 局部特征
  -> editor 表征
  -> 融合
  -> 四分类输出
```

5 句话解释 RACE：

```text
1. RACE 试图把 creator 和 editor 的痕迹分开看。
2. creator 更像决定文章整体逻辑和组织方式的人/模型。
3. editor 更像在局部表达、措辞、句子层面留下痕迹的人/模型。
4. RST 用来抓篇章逻辑结构，EDU 用来抓更细的局部单元。
5. 最后模型输出四类文本，而不是简单 AI/人类二分类。
```

### 板块 D：实验，4h

目标：看懂实验到底证明了什么，不需要把所有数字背下来。

| 小任务 | 时间 | 具体做法 | 输出 |
|---|---:|---|---|
| 数据集/任务 | 60min | 找数据来源、四分类标签、训练/测试设置 | 数据集说明 |
| baselines | 45min | 列出作者和哪些方法比 | baseline 表 |
| metrics | 30min | 搞清楚主要指标，不懂就查定义 | 指标解释 |
| 主结果表 | 60min | 只看 RACE 比谁强、强多少、大概在哪些类别强 | 主结论 3 条 |
| ablation/局限 | 45min | 看去掉模块后性能变化，顺手记风险 | 局限和疑问 |

实验总结表模板：

| 项目 | 你要填什么 |
|---|---|
| 数据集 | 用了什么数据，四类怎么构造 |
| baselines | 对比了哪些检测器/模型 |
| metrics | 主要看什么指标 |
| 主结果 | RACE 强在哪里 |
| ablation | 哪个模块最关键 |
| 局限 | 短文本、跨领域、强改写、数据构造是否有风险 |

### 板块 E：收束，4h

目标：最后要有能拿出来讲的东西。

| 小任务 | 时间 | 具体做法 | 输出 |
|---|---:|---|---|
| 一页中文总结 | 90min | 按“问题-方法-实验-局限”写 | 最终笔记 |
| 5 个问题 | 45min | 写真的没懂或想讨论的问题 | 问题清单 |
| 3 个亮点 | 45min | 写汇报时能讲的亮点 | 亮点清单 |
| 3 分钟口头版 | 60min | 练一次不看论文讲出来 | 口头稿 |

一页总结结构：

```text
1. 这篇论文解决什么问题：
2. 为什么普通 AI 文本检测不够：
3. 它怎么定义 creator/editor：
4. RACE 的核心思路：
5. 实验证明了什么：
6. 我觉得最重要的局限/疑问：
```
